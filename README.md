# CS 230 Portfolio Reflection

The Gaming Room was the client for this project. They wanted to expand their Android game, Draw It or Lose It, into a web-based application that could support multiple platforms and multiple users. The system needed to manage games, teams, and players while ensuring names remained unique and the application could run efficiently.

One thing I did well in this project was clearly organizing the client’s requirements and connecting them to the software design. I explained how the system architecture, domain model, and design patterns would support the application.

Working through the design document helped me understand how important it is to plan before coding. It allowed me to think through how the classes interact and how the system would function across different platforms.

If I could revise one part, I would improve the system architecture section by adding more detail about how the system communicates between the client, server, and database.

I focused on the user’s needs by making sure the system could support multiple users, teams, and games while staying efficient and scalable. Considering the user’s needs is important because the software must solve the client’s actual problem.

My approach to designing software was to first understand the requirements, then organize the system using object-oriented design. In the future, I would continue using design documents and diagrams to plan before coding.
